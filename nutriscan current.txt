# NutriScan: Current Codebase State & Architecture

**Last Updated**: May 1, 2026
**Project Status**: 95% Complete (MVP Phase)

## 1. Overview & Tech Stack
NutriScan is a React Native mobile application designed for users with specific health conditions (diabetes, hypertension, kidney disease, etc.) to scan food items and receive a health verdict (Safe, Caution, Avoid).

**Core Tech Stack:**
- **Framework**: React Native with Expo (Managed Workflow)
- **Navigation**: `expo-router` (File-based routing)
- **Language**: TypeScript
- **Styling**: Custom React Native StyleSheet with global Theme Tokens (no Tailwind)
- **State Management**: React Context (`AuthContext` + `ProfileContext`) + AsyncStorage + Supabase for persistence
- **Animations**: React Native `Animated` API & `react-native-reanimated`

## 3. Application Flow & Navigation Map

The routing is entirely handled by `expo-router` using contextual state checks in the Root layout and Splash screen.

### Authentication & Hydration Flow
1. **Splash Screen (`/`)**: Displays animated logo. After 2 seconds, checks `ProfileContext`.
2. **Auth Gateway**: If no authenticated user exists -> routes to `/(auth)/login`.
3. **Onboarding Gateway**: If user logged in but `hasOnboarded` is false -> routes to `/(onboarding)`.
4. **Main App**: If user is fully onboarded -> routes to `/(tabs)`.

### Core User Journey (The Scan Flow)
1. **Home (`/(tabs)/index`)**: User taps the large "Scan Food" CTA.
2. **Scan Hub (`/(tabs)/scan`)**: Placeholder for camera. User taps "Take Photo".
3. **Preview (`/scan-preview`)**: User reviews image, edits the recognized food name, selects Meal Type, and hits Analyze.
4. **Result (`/scan-result`)**: Displays the Verdict (Avoid, Caution, Safe). Warns about specific nutrients relative to their health conditions.
5. User taps "Add to Food Log" and is redirected to Home.

### Cross-Functional Flows
- **NutriBot Chat**: Accessible globally via the FAB on the Tab bar, from the Profile screen, or via Chat History.
- **Health Tips**: Carousel on the Home screen opens `/article/[id]` modal.
- **Reporting**: History tab (`/(tabs)/history`) links to `/health-report` for detailed analytics.

## 5. State Management & Data Layer

### `ProfileContext.tsx`
Handles the global persistence layer via `AsyncStorage`.
- `hasHydrated`: Ensures loading states don't flash before memory is read.
- `hasOnboarded`: Controls the gate between auth and the main app.
- `profile`: Stores the user's selected `conditions` and `goals`.

### `data/mockData.ts`
Because the app is not yet connected to a live backend (Gemini/Wikipedia), this file provides a robust, realistic data layer ensuring the UI works flawlessly:
- `MOCK_RESULT_AVOID`, `MOCK_RESULT_CAUTION`, `MOCK_RESULT_SAFE`
- `MOCK_TODAY_LOG` and `MOCK_WEEKLY` analytics
- `MOCK_TIPS` and `MOCK_ARTICLES` content
- `MOCK_HEALTH_REPORT` and `MOCK_CHAT_HISTORY`

## 6. Design System & Theming

The app completely bypasses third-party UI libraries in favor of a bespoke, scalable token system found in `constants/theme`.

- **Medical Teal**: Used as the primary brand color to evoke trust and health.
- **Semantic Verdicts**: 
  - `Safe`: Green (`#27AE60`)
  - `Caution`: Amber (`#F2994A`)
  - `Avoid`: Red (`#EB5757`)
- **Spacing/Radius**: Consistent `8px` grid system.
- **Components**: Everything is built on top of `AppScreen`, `TopBar`, and `Card` primitives, ensuring dark-mode readiness and consistent padding across devices.
